**Requisitos Funcionais (RF)**
- RF01 - Painel Administrativo: O sistema deve oferecer um dashboard para o hemocentro visualizar o status dos stocks de sangue.
- RF02 - Autenticação: O sistema deve possuir login para Administradores (Hemocentro) e Doadores.
- RF03 - Consulta de Aptidão: O doador deve poder visualizar no seu perfil a data em que estará apto para a próxima doação.
- RF04 - Mural de Urgências: A página inicial deve exibir um componente com os tipos sanguíneos em nível crítico.

**Requisitos Não Funcionais (RNF)**
- RNF01 - Interface SPA: O frontend deve ser desenvolvido como uma Single Page Application (ex: Angular) para garantir fluidez.
- RNF02 - API RESTful: A comunicação entre o frontend e o backend deve ser feita via protocolo HTTP seguindo padrões REST.
- RNF03 - Persistência: Os dados devem ser armazenados em um banco de dados relacional (ex: PostgreSQL ou MySQL).
